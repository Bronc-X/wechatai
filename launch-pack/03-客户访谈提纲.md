# 客户访谈提纲

## 访谈目标

确认客户的小程序是否适合做 Agent-ready 改造，并判断是否有付费价值。

重点不是问客户“要不要 AI”，而是判断：

- 他们是否有高价值交易流程。
- 用户是否需要填写大量字段。
- 当前转化是否有明显损耗。
- 业务数据是否可结构化。
- 是否有预算做小程序体验和转化改造。

## 基本信息

1. 你们的主要业务是什么？
2. 小程序承担什么角色：展示、预约、下单、支付、售后，还是客服？
3. 用户主要从哪里进入小程序：朋友圈、公众号、私域、广告、线下扫码、搜一搜？
4. 当前小程序月访问量、咨询量、预约量或订单量大概是多少？

## 核心业务流程

1. 用户完成一次下单 / 预约 / 咨询，一般要走多少步？
2. 哪些字段必须用户填写？
3. 哪些字段其实可以通过上下文、历史订单或默认值推断？
4. 用户最常卡在哪一步？
5. 客服最常帮用户做什么重复工作？

## 数据结构

1. 商品 / 服务是否有统一字段表？
2. 价格、库存、排班、门店状态是否实时？
3. 是否有订单、预约、退款、改约接口？
4. 哪些信息现在只写在图片、海报、富文本或人工话术里？

## 表单和体验

1. 用户最不愿意填的字段是什么？
2. 表单有没有被投诉过复杂或难懂？
3. 页面是否做过 A/B 测试或转化分析？
4. 当前小程序视觉是否符合品牌定位？
5. 有没有觉得“页面太旧 / 太丑 / 不像高端品牌”的问题？

## Agent-ready 场景

1. 如果用户只说一句话，你最希望小程序能帮他完成什么？
2. 哪些流程可以由 Agent 预填？
3. 哪些步骤必须用户亲自确认？
4. 哪些动作绝对不能让 Agent 自动执行？
5. 出错时希望系统如何兜底？

## 商业价值

1. 一个预约 / 订单的平均客单价是多少？
2. 一个有效线索值多少钱？
3. 当前从访问到预约 / 下单的转化率大概是多少？
4. 如果转化率提升 10%-20%，每月价值大概是多少？
5. 是否愿意为诊断报告或改造方案付费？

## 访谈后判断

适合进入付费改造的信号：

- 客单价高。
- 字段多。
- 用户经常找客服代填或咨询。
- 小程序转化率低。
- 价格、预约、服务项目可结构化。
- 客户愿意给真实页面和流程数据。

不适合优先做的信号：

- 客单价低且无预算。
- 业务完全非标，无法结构化。
- 小程序只是展示页，没有交易或预约。
- 客户只想“加个聊天机器人”。
- 客户不愿意提供流程和数据。
