# Agent-ready 小程序改造平台启动工作台

## 当前目标

用 4-6 周做出一个可演示、可销售、可试诊断真实小程序的启动版本。

第一版不追求接入微信官方 AI 入口，而是先完成三件事：

1. 建立 Agent-ready 诊断工具，用来获客、评分和输出改造建议。
2. 建立一个高利润垂直行业样板，用来演示“自然语言 -> 预约 / 下单 / 确认”的完整业务流。
3. 建立小程序 UI / 表单体验升级包，用来解决“页面丑、字段多、转化低、Agent 难识别”的问题。

## 产品边界

### 做

- 小程序 Agent-ready 评分。
- 商品 / 服务 / 预约 / 下单流程结构化诊断。
- 表单字段瘦身。
- 支付前确认和异常恢复路径设计。
- 小程序 UI 和移动端体验升级建议。
- 垂直行业业务流样板。
- 客户访谈、诊断报告、改造报价包。

### 暂不做

- 不承诺接入微信官方 AI 入口。
- 不做真实支付自动化。
- 不做强监管医疗诊断结论。
- 不自研或微调大模型。
- 不做通用小程序生成器。

## 推荐第一行业

第一条垂直样板建议选“医美 / 口腔咨询预约”。

理由：

- 客单价高，商家有预算。
- 当前咨询和预约字段多，Agent 有明显减负价值。
- 可以先做咨询、预约、支付定金，不碰医疗诊断结论。
- 业务字段可结构化，例如项目、门店、医生、时间、价格、禁忌、注意事项。

备选行业：

- 家政 / 本地上门服务。
- 汽车保养 / 维修报价。
- 高端餐饮 / 茶饮连锁。

## MVP 成功标准

### 工具侧

- 能录入一个小程序的行业、页面、流程和问题。
- 能按 9 个维度生成 Agent-ready 分数。
- 能输出高优先级改造建议。
- 能把报告用于一次客户沟通。

### 垂直样板侧

- 能演示用户一句话提出需求。
- 能拆解成结构化参数。
- 能查询候选服务方案。
- 能生成支付前确认信息。
- 能覆盖至少 5 类失败状态。

### 销售侧

- 能完成 3-5 个真实小程序试诊断。
- 能拿到至少 1 个愿意深入访谈或付费改造的客户。
- 能沉淀第一版报价包。
