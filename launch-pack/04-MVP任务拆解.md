# 4-6 周 MVP 任务拆解

## 第 1 周：资产固化与样板确认

目标：

- 明确产品边界。
- 完成诊断评分表。
- 确认第一个垂直样板。
- 建立销售沟通材料。

任务：

1. 固化 PRD。
2. 完成 Agent-ready 评分表。
3. 完成医美 / 口腔样板。
4. 完成客户访谈提纲。
5. 做一个静态诊断原型。
6. 准备 3 个真实小程序试诊断名单。

验收：

- 能拿着材料和客户讲清楚“不是做 AI 助手，而是做 Agent-ready 转化改造”。

## 第 2 周：诊断工具 MVP

目标：

- 做出可用的诊断工具第一版。

任务：

1. 支持录入小程序名称、行业、核心目标。
2. 支持 9 个维度手动评分。
3. 自动计算总分和分级。
4. 自动生成高优先级建议。
5. 支持导出 Markdown 报告。
6. 增加 UI / 表单体验评分。

验收：

- 能对一个真实小程序完成 10 分钟诊断。

## 第 3 周：垂直样板 demo

目标：

- 做出医美 / 口腔预约 demo 流程。

任务：

1. 定义行业 schema。
2. 定义自然语言解析参数。
3. 定义预约候选方案结构。
4. 定义支付前 / 预约前确认页。
5. 定义异常恢复路径。
6. 做一段可演示对话脚本或交互原型。

验收：

- 能演示“用户一句话 -> 候选方案 -> 用户确认 -> 预约成功 / 异常恢复”。

## 第 4 周：客户试诊断

目标：

- 用真实客户验证诊断价值。

任务：

1. 找 3-5 个小程序做免费或低价诊断。
2. 输出每个客户的诊断报告。
3. 记录客户最关心的问题。
4. 总结共性改造点。
5. 形成第一版报价包。

验收：

- 至少 1 个客户愿意继续深聊改造。

## 第 5-6 周：付费交付准备

目标：

- 把一次咨询变成可交付服务。

任务：

1. 完成标准诊断报告模板。
2. 完成行业改造包目录。
3. 完成小程序 UI 升级包目录。
4. 完成报价梯度。
5. 完成交付周期和验收标准。
6. 准备对外介绍页。

验收：

- 能正式销售“诊断包 + 垂直改造包 + 持续优化订阅”。

## 推荐报价结构

### 入门诊断

交付：

- Agent-ready 评分。
- UI / 表单体验评分。
- 10 条关键改造建议。

适合：

- 获客和低价转化。

### 标准诊断包

交付：

- 完整诊断报告。
- 页面 / 字段 / 流程问题清单。
- 改造优先级。
- 1 个核心流程改造方案。

适合：

- 有明确转化问题的小程序商家。

### 垂直改造包

交付：

- 行业 schema。
- 页面和表单改造。
- 预约 / 下单 / 支付确认流程。
- 异常恢复路径。
- 上线验证。

适合：

- 医美、口腔、家政、汽车服务等高价值客户。

### 持续优化订阅

交付：

- 月度评分。
- 转化卡点分析。
- 新页面审查。
- 失败状态复盘。
- 行业 benchmark。
