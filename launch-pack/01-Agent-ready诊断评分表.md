# Agent-ready 诊断评分表

## 使用方式

本评分表用于判断一个小程序是否适合被微信 AI / Agent 调用。每个维度 0-5 分，总分 45 分。

建议分级：

- 0-15 分：暂不适合 Agent 调用，先做基础结构和流程整改。
- 16-30 分：具备部分基础，可做重点流程改造。
- 31-40 分：较适合 Agent 调用，可进入垂直样板和试点。
- 41-45 分：高度 Agent-ready，可进一步做转化优化和行业 benchmark。

## 评分维度

### 1. 业务数据结构化

检查：

- 商品 / 服务是否有标准名称。
- 规格、价格、库存、门店、时间、服务条件是否机器可读。
- 关键信息是否藏在图片、富文本或人工备注里。
- 是否支持按价格、距离、时间、偏好等条件筛选。

评分：

- 0 分：核心信息只能靠人看页面理解。
- 1 分：有少量结构化字段，但不覆盖关键交易信息。
- 3 分：核心字段部分结构化，但实时性和完整性不足。
- 5 分：核心交易字段完整结构化，Agent 可直接读取。

### 2. 用户意图映射能力

检查：

- 自然语言需求能否映射到业务字段。
- 是否支持口语表达、同义词、模糊条件。
- 是否能处理预算、距离、时间、偏好、禁忌等约束。

示例：

“这周末想洗牙，预算 500 以内，离我近一点”应能映射到项目、预算、时间、位置和门店。

### 3. 关键流程短链路

检查：

- 查询、筛选、选择、确认、支付、取消路径是否清楚。
- 是否有重复页面、无意义弹窗、强制跳转。
- 是否可把多页操作压缩成 Agent 可执行动作。

### 4. 实时状态可信

检查：

- 价格是否实时。
- 库存 / 号源 / 排班是否实时。
- 门店营业状态是否实时。
- 优惠券和活动规则是否准确。

### 5. 登录与授权边界

检查：

- 登录是否只在必要节点出现。
- 定位、手机号、联系人、健康信息等权限是否解释清楚。
- 授权失败后是否有可继续路径。

### 6. 支付前确认安全

检查：

- 支付前是否展示商品 / 服务内容、金额、优惠、门店、时间、地址、取消规则。
- 是否有明确的用户确认节点。
- 是否防止重复提交和误下单。

### 7. 异常状态恢复

检查：

- 无库存、预约冲突、支付失败、优惠券不可用、地址超范围时是否有下一步。
- 用户中途退出后是否能恢复。
- 是否有人工客服兜底。

### 8. 售后与订单状态

检查：

- Agent 能否查询订单。
- 是否支持改约、取消、退款、联系客服。
- 订单状态是否清晰。

### 9. UI / 视觉 / 表单体验

检查：

- 页面是否可信、美观、移动端友好。
- 信息层级是否清楚。
- 表单字段是否过多。
- 按钮和 CTA 是否明确。
- 是否存在文本溢出、点击区域过小、加载状态缺失。
- 页面结构是否影响 Agent 识别。
